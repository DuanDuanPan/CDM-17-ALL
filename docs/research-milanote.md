## 产品概览（公开能力）
- 实时协同：同板多人同步、光标可见、评论线程，@ 提及即通知；免费版也可协同。（来源待补充）
- 通知：应用内变更提醒 + 每日邮件摘要；@mention 触发通知。（来源待补充）
- 导出：板级导出 PDF（标准/大）、PNG、Word、Markdown、纯文本；可生成 ZIP 抓取板内文件。（来源待补充）
- 模板：官方模板库覆盖脑图/头脑风暴/项目计划/旅程地图等。（来源待补充）
- 权限分享：板级邀请编辑或只读链接，可查看访问者；缺乏字段/附件粒度。（来源待补充）
- 跨端：Web + iOS/Android；移动端“不支持离线，需联网”，表格编辑受限。（来源待补充）

## 定价与限制（2025-12-12）

## 已知短板 / 负向信号
- 离线缺失：官方 FAQ 明确移动端需联网；社区反馈偶尔短暂可访问但又被收回，稳定性差。（来源待补充）
- API 不公开：公开渠道标注“无 API”。（来源待补充）
- 批量/层级导出受限：ZIP 只抓当前板文件，不含嵌套子板，用户抱怨备份困难。（来源待补充）
- 版本/历史：未公开板级版本时间线；协同只显示最近访问与编辑者，无回滚。（来源待补充）
- 性能稳定性：社区讨论大板/离线尝试易崩溃或 503，担忧大规模使用可靠性。（来源待补充）
- 免费额度低：100 卡/10 文件被频繁吐槽，促使付费。（来源待补充）

## 差异化对照（与本项目 SoT 目标）
- 权限/合规：Milanote 仅板级分享，缺失密级、字段/附件级、动态水印、审计导出 → 可作为核心卖点。
- 执行/任务：任务卡片简单，缺少 FS/SS/FF/SF 依赖、审批驱动推演；无甘特/时间轴联动 → 我们可做“脑图→任务编排”。
- 版本/时间线：缺乏板级版本时间线与回滚；我们规划多版本+时间线。
- 血缘/跨图：无跨板血缘与数据流；我们可做跨图引用、血缘可视化、下钻回链。
- 通知节流：仅基础提醒，无“5 分钟去重+汇总”；可作为噪音治理差异化。
- 性能指标：未公开大图指标；社区抱怨卡顿/宕机；我们承诺 1k+ 节点、50 并发无降级。
- 开放性：无 API/批量导出；我们可提供 API、导出与集成（Webhook/IM/CI）。

## 机会与对手动作假设
- 近期未见离线/自托管/API 路线；仍以云 SaaS 为主 → 可抢涉密/内网市场。
- 价格对 SMB 友好但免费额度低 → 可提供更大试用或按席位灵活包。
- 宕机与备份争议提示稳定性焦虑 → 可提供状态页、SLA、全局导出/备份与只读降级模式。

## 结论 & 建议
- 定位：将 Milanote 视为“创意视觉工作区”，非工程执行 SoT。
- 竞争策略：
  1) 强调安全/密级/字段附件权限 + 动态水印 + 审计导出。
  2) 打通“脑图→任务/甘特/审批推演”形成闭环。
  3) 公布可验证的性能基线（1k+节点、50 并发）与观测指标。
  4) 提供 API、批量导入导出、跨图血缘；预留国产化/私有化。
  5) 通知节流与订阅策略作为噪音治理卖点。
