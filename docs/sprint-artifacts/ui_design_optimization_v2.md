## 设计改进点 (Design Highlights)

### 1. 节点规格统一化 (Standardization)
- **问题**: 原设计中普通节点（如底部文本节点）宽度随内容变化，导致布局杂乱且浪费横向空间。
- **优化**: 强制所有节点（包括普通节点和卡片节点）使用**固定宽度**（建议 220px - 240px）。
    - 既保证了视觉对齐的整洁感。
    - 又解决了"两遍空白区域"（Padding）不可控的问题，将内容限制在固定容器内，文本居中或左对齐截断。

### 2. 信息密度极大提升 (High Density)
- **问题**: 原卡片节点（蓝/绿/黄）内部留白过多，导致单屏可见节点数量少。
- **优化**: 采用**紧凑布局 (Compact Mode)**。
    - **内边距 (Padding)**: 从原来的 `16px/20px` 压缩至 `8px/12px`。
    - **行高控制**: 标题与图标单行显示，元数据（Status/ID）单行显示。
    - **去重**: 移除多余的装饰性空白。
- **结果**: 单屏可见节点数量提升约 **150% - 200%**。

### 3. 视觉分层与细节 (Visual Hierarchy)
- **普通节点 (Simple Nodes)**: 采用扁平的矩形设计，高度压缩，去除多余装饰，仅保留必要的连接点功能。
- **卡片节点 (Cards)**:
    - 头部：Icon + Title 紧凑排列。
    - 底部：Tag + ID 左右对齐，利用角落空间。
    - 连线：使用平滑的贝塞尔曲线，配合网格背景，提升专业工程软件的质感。
